# New Life ID — Student

**FSD:** §20

## 1. Purpose

Students see their New Life ID so they can include it on retail shipments and reference it with support.

## 2. User stories

| ID | Story |
|----|-------|
| S-ID1 | As a student, I see my New Life ID on my dashboard and profile. |
| S-ID2 | As a student, I am reminded to put my name + New Life ID on retail packages. |

## 3. Display locations

| Location | Format |
|----------|--------|
| Student dashboard | Welcome or profile card |
| My Profile | Prominent read-only field |
| Retail package form | Helper text / acknowledgement |
| Support ticket | Auto-attached to submission |

## 4. Business rules

- ID assigned at account provisioning; never user-editable
- Shown in copy-paste friendly format

## 5. Acceptance criteria

- [ ] New Life ID visible on dashboard after onboarding.
- [ ] ID visible on profile page.
- [ ] Retail package flow references ID in acknowledgement text.
